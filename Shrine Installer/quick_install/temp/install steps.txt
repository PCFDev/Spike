#NOTE: i2b2 Steps
1. run the configure_hive_db_lookups.sql on i2b2hive database
2. update the ont-ds.xml on i2b2 server jboss deployments
3. run the configure_pm.sql on i2b2pm database
4. run these scripts on i2b2metadata database
  ontology_create_user.sql
  ontology_create_tables.sql
  ontology_table_access.sql
  https://open.med.harvard.edu/svn/shrine/releases/1.18.2/onotlogy/SHRINE_Demo_Downloads/ShrineDemo.sql

#NOTE: shrine steps
1. run mssql.sql on master database with server admin rights
2. run adapter.sql on shrine database
3. run hub.sql on shrine database
4. run create_broadcaster_audit_table.sql on shrine database
5. install tomcat
6. configure keystore
  a. generate cer file
  b. move it to shrine's tomcat folder
7. update the shrine.conf file (/opt/shrine/tomcat/lib/shrine.conf)
8. update firewall to allow shrine ports
9. download shrine war files from http://repo.open.med.harvard.edu/nexus/content/groups/public/net/shrine/
  a. shrine-war-${SHRINE_VERSION}.war => shrine.war
  b. shrine-proxy-${SHRINE_VERSION}.war => shrine-proxy.war
10. move the war files to the webapps folder of tomcat
11. download the shrine-webclient from SVN
  a. svn export --quiet https://open.med.harvard.edu/svn/shrine/releases/1.18.2/code/shrine-webclient/ shrine-webclient
12. download the adapter-mappings.xml file
  a. https://open.med.harvard.edu/svn/shrine/releases/${SHRINE_VERSION}/onotlogy/AdapterMappings_i2b2_DemoData.xml
13. copy the adaptermappings.xml file to the /opt/shrine/tomcat/lib folder
14. setup keystore and ports in tomcat_server.xml and copy it to /opt/shrine/tomcat/conf/server.xml
15. make a directory at /opt/shrine/tomcat/conf/Catalina/localhost
16. update shrine.xml and copy it to /opt/shrine/tomcat/conf/Catalina/localhost
17. backup and update shrine-webclient/i2b2_config_data.js shrine-webclient/i2b2_config_data.js
18. backup and update shrine-webclient/js-i2b2/cells/SHRINE/cell_config_data.js
19. copy shrine-webclient to /opt/shrine/tomcat/webapps/
